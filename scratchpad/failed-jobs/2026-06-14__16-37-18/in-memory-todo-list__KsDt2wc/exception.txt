# In-Memory Todo List App (Reflex)

## Background

Build a small single-page **Reflex** application that implements an in-memory todo list. There is no database involved — all data lives in the Reflex `State`, synchronized to the browser over the Reflex websocket protocol. The page must dynamically render the items using `rx.foreach`, expose an event handler that adds new items, an event handler that removes items by index, and a cached computed var that reports the total number of items.

## Requirements

- A Reflex `State` subclass that exposes at minimum:
  - A synchronized base var `new_item: str` (the current value of the input field), defaulting to the empty string.
  - A synchronized base var `items: list[str]` (the list of todo items), defaulting to an empty list.
- Two (or more) event handlers on that State:
  - One that **adds** the current value of `new_item` to the `items` list and then clears `new_item` back to the empty string.
  - One that **removes** a single item from `items` by its integer index.
- A **cached computed var** named `total` on the same State that returns `len(self.items)` (i.e., the current item count).
- An index page at route `/` that:
  - Renders an input control bound to `new_item` (so that typing updates the State var).
  - Renders an **"Add"** button (literal label `Add`) that triggers the add event handler.
  - Renders the items by calling `rx.foreach(State.items, render_item)` (any function name is allowed for `render_item`). Each rendered row must display the item text and contain a **"Remove"** button (literal label `Remove`) that triggers the remove-by-index event handler.
  - Displays the total in the literal format `Total: N`, where `N` is rendered from the `total` computed var (e.g., `rx.text("Total: ", State.total)`).

## Implementation Hints

- Follow the project setup flow from the research plan: `uv init`, `uv add reflex`, `uv run reflex init --template blank`, and then run with `uv run reflex run --loglevel info`.
- Bind the input value to `new_item` with `on_change` (or an equivalent `set_*` event trigger) so the State stays in sync as the user types.
- For the remove handler, pass the index from the `rx.foreach` render function (the function may take `(item, index)`).
- Use `@rx.var(cache=True)` to declare the cached computed var.
- After you have finished developing, **kill all background servers** you started (e.g. any `uv run reflex run`). The verifier starts the server itself.
