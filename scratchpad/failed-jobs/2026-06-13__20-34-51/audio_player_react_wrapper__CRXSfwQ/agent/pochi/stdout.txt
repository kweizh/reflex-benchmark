# Wrap `react-h5-audio-player` as a Reflex NoSSRComponent with Playlist State ## Background Reflex compiles Python UI into a Next.js / React frontend. Many useful UI libraries are only available on npm. The [`react-h5-audio-player`](https://github.com/lhz516/react-h5-audio-player) package ships a `<AudioPlayer />` React component that touches `window.Audio`, so it must be wrapped as a Reflex `NoSSRComponent` (see [Wrapping React Overview](https://reflex.dev/docs/wrapping-react/overview/)). Your job is to wrap it, expose two props and three event triggers, and drive it from a small playlist state machine. ## Requirements - Build a Reflex app named `audio_app` at `/home/user/audio_app` using `uv` per the rules in the project plan (use `uv init`, `uv add reflex`, `uv run reflex init --template blank`). - Author a Python wrapper for `react-h5-audio-player` as a `NoSSRComponent`. - The wrapper must expose **exactly** these two props as Reflex Vars: `src: rx.Var[str]` and `autoplay: rx.Var[bool]`. The Python attribute names must literally be `src` and `autoplay`; you are responsible for mapping them onto the underlying React props (the React component uses `src` and `autoPlay`). - The wrapper must declare three event triggers ‚Äî `on_play`, `on_pause`, `on_ended` ‚Äî each defined via `rx.EventHandler[...]` with a serializer lambda that produces **zero** positional arguments (i.e. the event handler signature on the Python side takes no extra args). - Build a single page at route `/` that displays: - The wrapped audio player driven by the current playlist track. - A list of exactly 3 selectable track entries; clicking a track sets that track as the current one. - Implement a `State` class (subclass of `rx.State`) with: - A backend list of exactly 3 tracks (each with a `title` and a `src` URL ‚Äî use local placeholder paths such as `/tracks/track1.mp3`, etc.; do **not** read any external service). - An `int` Base Var named `current_index` representing the active track. - An event handler that updates `current_index` when a track is clicked. - An event handler wired to the wrapper's `on_ended` trigger that advances `current_index` by one and **loops back to 0** after the last track. - The page must bind the wrapper's `src` Var to the current track's URL so that switching the index swaps the audio source. ## Implementation Hints - Read the React component's prop and event names from the upstream README before wiring up the wrapper. - `NoSSRComponent` lives in `reflex.components.component`. - `react-h5-audio-player` uses a **default** export; consider the `is_default` class attribute. - Reflex auto-converts snake_case Python prop names to camelCase on the React side, but a single word like `autoplay` will not magically become `autoPlay`. Look into prop renaming mechanisms (e.g. `_rename_props`) when the names diverge. - The library ships a separate CSS file (`react-h5-audio-player/lib/styles.css`). You will need to inject it into the compiled output (e.g. via `add_imports` / `_get_custom_code` / `add_style` ‚Äî pick whichever the docs you read recommend). - For event triggers that should serialize zero arguments, the lambda used in `rx.EventHandler[...]` takes no parameters and returns `[]`. - All Python dependencies must be managed with `uv`. After installing, generate `requirements.txt` (e.g. `uv pip freeze > requirements.txt`) at the project root so the project remains Reflex-Cloud-deployable per `plan.md`. - **Background servers MUST be killed before finishing the task.** Do not leave `reflex run`, `reflex export`, or any other Reflex process listening on ports 3000 or 8000. The verifier will start its own short-lived processes. ## Acceptance Criteria - Project path: `/home/user/audio_app` - Start command: `uv run reflex run --loglevel debug` - Port: frontend on `3000`, backend on `8000` - A file `requirements.txt` exists at the project root listing `reflex` (any version `>=0.6.6`). - A Python module inside the project defines a class that subclasses `reflex.components.component.NoSSRComponent` (directly or indirectly) and: - Sets `library` to a string whose first token (before any `@version` suffix) is exactly `react-h5-audio-player`. - Sets a `tag` attribute (any non-empty string). - Sets `is_default = True`. - Declares a class attribute `src` typed as `rx.Var[str]`. - Declares a class attribute `autoplay` typed as `rx.Var[bool]`. - Declares three event triggers `on_play`, `on_pause`, `on_ended`, each annotated as `rx.EventHandler[...]` with a serializer lambda that takes no arguments and returns an empty list (`[]`). - A `State` class (subclass of `rx.State`) exposes: - A list-typed Base Var holding exactly 3 track entries. - An `int` Base Var named `current_index` defaulting to `0`. - An event handler that, given an integer index, sets `current_index` to that integer. - An event handler bound to the wrapper's `on_ended` trigger that advances `current_index` to `(current_index + 1) % 3`. - Building the frontend with `uv run reflex export --frontend-only --no-zip` succeeds, and the resulting compiled assets reference the string `react-h5-audio-player` (i.e. the wrapper survived bundling). - After the task is complete, no Reflex process is running on ports 3000 or 8000. ## Notes - Do not set any external environment variables for the playlist; the 3 placeholder URLs are part of the source code. - Do not deploy to Reflex Cloud; only local export is required.
