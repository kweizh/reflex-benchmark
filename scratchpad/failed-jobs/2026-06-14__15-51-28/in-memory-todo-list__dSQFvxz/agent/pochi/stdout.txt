# In-Memory Todo List App (Reflex) ## Background Build a small single-page **Reflex** application that implements an in-memory todo list. There is no database involved — all data lives in the Reflex `State`, synchronized to the browser over the Reflex websocket protocol. The page must dynamically render the items using `rx.foreach`, expose an event handler that adds new items, an event handler that removes items by index, and a cached computed var that reports the total number of items. ## Requirements - A Reflex `State` subclass that exposes at minimum: - A synchronized base var `new_item: str` (the current value of the input field), defaulting to the empty string. - A synchronized base var `items: list[str]` (the list of todo items), defaulting to an empty list. - Two (or more) event handlers on that State: - One that **adds** the current value of `new_item` to the `items` list and then clears `new_item` back to the empty string. - One that **removes** a single item from `items` by its integer index. - A **cached computed var** named `total` on the same State that returns `len(self.items)` (i.e., the current item count). - An index page at route `/` that: - Renders an input control bound to `new_item` (so that typing updates the State var). - Renders an **"Add"** button (literal label `Add`) that triggers the add event handler. - Renders the items by calling `rx.foreach(State.items, render_item)` (any function name is allowed for `render_item`).
