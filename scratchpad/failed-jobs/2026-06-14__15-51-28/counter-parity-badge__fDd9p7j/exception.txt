# Reflex Counter with Parity Badge

## Background
Build a small Reflex web application that demonstrates core Reflex primitives: a `rx.State` subclass with base vars, event handlers that mutate state, a **cached computed var** (`@rx.var(cache=True)`), and conditional UI rendering with `rx.cond`. The app renders a counter and a color-coded badge that indicates whether the current count is even or odd.

## Requirements
- Create a Reflex application at `/home/user/myproject` initialized non-interactively with the `blank` template using `uv` and the Astral `uv run reflex` CLI (see plan.md / Reflex Quickstart docs).
- Define a `rx.State` subclass with a synchronized base var `count: int = 0`.
- Define three event handlers on the state:
  - One that increments `count` by 1.
  - One that decrements `count` by 1.
  - One that resets `count` to `0`.
- Define a **cached** computed var named `parity` that returns the string `"even"` when `count % 2 == 0`, otherwise `"odd"`. The computed var must be cached (i.e. use `@rx.var(cache=True)`).
- Register a single page at the route `/` whose layout includes:
  - A heading using `rx.heading` that displays text in the form `Count: <count>` where `<count>` is bound to `State.count`.
  - Three buttons with the visible labels `Increment`, `Decrement`, and `Reset`, each wired to the matching event handler.
  - A `rx.badge` whose visible text is the value of the `parity` computed var, and whose `color_scheme` is selected via `rx.cond` so the badge is **green** when parity is `"even"` and **red** when parity is `"odd"`.
