# Reflex App: OAuth2 PKCE Login with FastAPI Custom Routes ## Background Build a Reflex application that mounts FastAPI routes via `api_transformer` to implement an OAuth2 PKCE (RFC 7636) authorization-code flow against a local **mock** OAuth provider that is hosted inside the **same** Reflex/FastAPI app. The Reflex UI exposes a "Sign in" button that starts the flow, and the callback exchanges the authorization code for an access token, persists the token in a client-side `rx.Cookie`, and mirrors it into a backend-only state var so authenticated server-side requests can use it. ## Requirements * Build the Reflex project at `/home/user/oauth_app` and manage Python dependencies with `uv` (per the framework convention). * Define a top-level FastAPI app and mount it on Reflex via `rx.App(api_transformer=fastapi_app)`. * Implement a **mock OAuth provider** with the following endpoints under `/auth/*` (mounted via the FastAPI app): * `GET /auth/authorize` — the provider's authorization endpoint. * `POST /auth/token` — the provider's token endpoint (form-encoded body). * Implement an **application-side** flow with these endpoints under the same FastAPI app: * `GET /auth/start` — initiates the PKCE flow: generates `code_verifier`, derives `code_challenge` using **S256**, generates an opaque `state`, stores the verifier server-side keyed by `state`, and redirects (HTTP 302) to `/auth/authorize` with all required PKCE/OAuth2 query params. * `GET /auth/callback` — receives `code` and `state`, looks up the matching `code_verifier`, exchanges the code at `/auth/token`, sets a non-HttpOnly `access_token` cookie with `Path=/`, and redirects (HTTP 302) to `/`. * Implement a protected API endpoint `GET /api/me` that returns 200 with the demo user identity only when a valid `Authorization: Bearer <token>` header is present, and 401 otherwise. * Implement the Reflex UI: * Route `/` shows a heading containing the text `Sign in` and a clickable element (link or button) that navigates the browser to `/auth/start`. * The application state must declare an `access_token` field using `rx.Cookie` (so it is synchronized to/from the browser cookie of the same name) **and** a backend-only state var (name starting with `_`) that mirrors the same value for use in authenticated server-side requests. * Persist the generated credentials and demo identity to a known location so an external client can drive the flow without scraping source code (see Acceptance Criteria). ## Implementation Hints * Use `secrets.token_urlsafe(...)` for the verifier/state. Use Python's `secrets` module. * Keep the per-flow `state → code_verifier` map and the per-code `code → {code_challenge, redirect_uri}` map in module-level dictionaries (single-process server is fine for this task). * Reuse the same FastAPI app for both the mock provider and the application-side routes; mount it via `api_transformer`. * The protected `/api/me` route should reject any request whose token does not match the most recently issued token. * For the Reflex UI, you only need a minimal `/` page; no styling is required.
