# Markdown Editor with Live HTML Preview (Reflex) ## Background Build a single-page Reflex application that gives the user a Markdown editing experience with a live HTML preview. The page is split into two columns: an editable Markdown source area on the left, and a live-updating HTML preview on the right. A toolbar above (or beside) the textarea offers `Bold`, `Italic`, and `Code` buttons that wrap the user's current text selection with the appropriate Markdown markers. A counter element shows the live word and character counts of the source. The Markdown-to-HTML conversion must happen on the backend in a cached computed var, and the HTML must be injected into the page via `rx.html`. ## Requirements - A Reflex project at `/home/user/myproject` initialized with `uv` (per the framework's recommended setup) and a Reflex app exposing a page at `/`. - A single Reflex `State` subclass that owns at minimum: - `source: str` — the current Markdown source (controlled value of the textarea). - `selection_start: int` and `selection_end: int` — the cursor's current selection range inside the textarea, updated from a textarea event handler whenever the user changes the selection or types. - A cached computed var (decorated with `@rx.var(cache=True)`) named `preview_html` that converts `self.source` into HTML using the `markdown-it-py` library and returns the HTML string. - Cached computed vars (decorated with `@rx.var(cache=True)`) named `word_count` and `char_count` that return the number of words and the number of characters in `self.source`. - Event handlers `wrap_bold`, `wrap_italic`, and `wrap_code` that mutate `self.source` by wrapping the substring from `selection_start` to `selection_end` with the Markdown markers `**…**`, `*…*`, and `` `…` `` respectively. When the selection is empty (`selection_start == selection_end`), insert the marker pair at the cursor position so the cursor sits between the markers. - The page layout must include the following DOM hooks (these IDs are how the verifier locates each piece): - A `<textarea>` element bound to `State.source` with the HTML id `md-source`. - A preview container element with HTML id `md-preview` whose inner HTML is produced by `rx.html(State.preview_html)`. - A counter element with HTML id `md-counter` whose displayed text follows the format `Words: <word_count> | Characters: <char_count>` (no other text in this element). - Three toolbar buttons with HTML ids `btn-bold`, `btn-italic`, and `btn-code` that trigger the corresponding wrap event handlers. - The Markdown converter must be instantiated using `markdown-it-py`'s `MarkdownIt` class (any preset — `commonmark` is fine — as long as it produces standard HTML such as `<h1>`, `<p>`, `<strong>`, `<em>`, `<code>`, `<ul>`, `<li>` for the corresponding Markdown constructs). -
