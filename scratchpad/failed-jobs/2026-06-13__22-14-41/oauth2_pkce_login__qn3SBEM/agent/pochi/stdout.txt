Requirements * Build the Reflex project at `/home/user/oauth_app` and manage Python dependencies with `uv` (per the framework convention). * Define a top-level FastAPI app and mount it on Reflex via `rx.App(api_transformer=fastapi_app)`. * Implement a **mock OAuth provider** with the following endpoints under `/auth/*` (mounted via the FastAPI app): * `GET /auth/authorize` — the provider's authorization endpoint. * `POST /auth/token` — the provider's token endpoint (form-encoded body). * Implement an **application-side** flow with these endpoints under the same FastAPI app: * `GET /auth/start` — initiates the PKCE flow: generates `code_verifier`, derives `code_challenge` using **S256**, generates an opaque `state`, stores the verifier server-side keyed by `state`, and redirects (HTTP 302) to `/auth/authorize` with all required PKCE/OAuth2 query params. * `GET /auth/callback` — receives `code` and `state`, looks up the matching `code_verifier`, exchanges the code at `/auth/token`, sets a non-HttpOnly `access_token` cookie with `Path=/`, and redirects (HTTP 302) to `/`. * Implement a protected API endpoint `GET /api/me` that returns 200 with the demo user identity only when a valid `Authorization: Bearer <token>` header is present, and 401 otherwise. * Implement the Reflex UI: * Route `/` shows a heading containing the text `Sign in` and a clickable element (link or button) that navigates the browser to `/auth/start`. * The application state must declare an `access_token` field using `rx.Cookie` (so it is synchronized to/from the browser cookie of the same name) **and** a backend-only state var (name starting with `_`) that mirrors the same value for use in authenticated server-side requests. * Persist the generated credentials and demo identity to a known location so an external client can drive the flow without scraping source code (see Acceptance Criteria). ##
