A loading spinner is bound to a backend-only flag while the stream is running. The interesting part is on the server: the streaming work runs in a Reflex background event, holds the state lock only when mutating shared state, and cooperatively observes a cancellation flag. ## Requirements * Reflex application at `/home/user/streaming_chat` initialized with the `blank` template and managed by `uv`. * A `StreamingChatState` class (a subclass of `rx.State`) that exposes: * The currently rendered assistant response, * A loading indicator surfaced through a Reflex computed/base var, * Backend-only flags (names starting with `_`) for the in-flight streaming status and the cooperative cancel signal. * An async event handler decorated with `@rx.event(background=True)` that simulates a streaming LLM reply token-by-token. The Reflex background event and the FastAPI endpoints **must share the same underlying streaming controller** so that triggering the stream from the API exercises the same cancellation/lock semantics that the UI does. * The server is started by the verifier in **backend-only** mode on port 8000. * After the test runs, **all background servers must be terminated** (the verifier starts its own server; leftover servers will cause port conflicts). ## Implementation Hints * The friction you should design around is `ImmutableStateError`: a background task cannot write `self.<var>` outside `async with self`.
