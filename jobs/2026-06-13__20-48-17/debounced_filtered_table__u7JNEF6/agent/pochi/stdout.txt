# Debounced Multi-Filter Data Table (Reflex) ## Background Build a Reflex web application that renders a searchable, filterable, sortable data table backed by a SQLite database. The table lists a catalog of `Product` rows that the application must seed deterministically at startup. The text-search filter must be debounced on the client (so rapid typing produces a single query), and the recomputation of filter results must run as an async background event using SQLModel `where`/`order_by` clauses against `rx.asession`. Subsequent starts must not duplicate rows. * Build a single page mounted at `/` containing the following reactive filter controls bound to State vars: * Text search input wrapped in `rx.debounce_input(...)` with `debounce_timeout=300` (milliseconds). * A category dropdown / select offering the literal value `All` plus each of the six category names. * Two numeric inputs for `min_price` and `max_price`. * A checkbox labeled "In stock only" bound to a boolean state var. * A sort-column selector and a sort-direction (asc/desc) selector. * The State must expose at least the following synchronized base vars: `result_count: int`, `filtered: list[Product]`, and the individual filter/sort vars listed above. * Recomputation of `filtered` and `result_count` MUST happen inside an `@rx.event(background=True)` async handler that opens an `rx.asession()` and constructs a SQLModel `select` with `.where(...)` clauses and an `.order_by(...)` clause. All mutations to state must happen inside an `async with self:` block. The background event must be triggered whenever any filter or sort var changes. * The results table must be rendered with `rx.foreach` iterating over `State.filtered`, displaying at least `id`, `name`, `category`, `price`, and `in_stock` per row. Total rows after seeding: 240. Idempotency: if the `product` table already contains rows when the application starts, do NOT re-seed. ### Filter & Sort Semantics (used by both the State background event and `/api/filter`) * `search` (string, default ""): case-insensitive substring match against `Product.name`. Empty string means no constraint. * `category` (string, default "All"): when value equals `All` or empty string, no constraint; otherwise equality match against `Product.category`. * `min_price` (float or null, default null): when provided, `Product.price >= min_price`. * `max_price` (float or null, default null): when provided, `Product.price <= max_price`. * `in_stock_only` (bool, default false): when true, restrict to rows with `in_stock == true`. * `sort_by` (string, default `id`): one of `id`, `name`, `price`, `category`. * `sort_dir` (string, default `asc`): one of `asc`, `desc`. ### API Contract for `GET /api/filter` Query parameters (all optional): * `search`: string * `category`: string * `min_price`: float * `max_price`: float * `in_stock_only`: `true` or `false` (case-insensitive) * `sort_by`: `id` | `name` | `price` | `category` * `sort_dir`: `asc` | `desc` Response: `200 OK` with JSON body shaped exactly like: ``` { "result_count": , "filtered": [ { "id": <int>, "name": <str>, "category": <str>, "sku": <str>, "price": <float>, "in_stock": <bool> }, ... ] } ``` The length of `filtered` MUST equal `result_count`.
